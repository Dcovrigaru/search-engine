About This Search Engine

Overview

This is a custom-built web search engine that implements both content-based and link-based ranking algorithms. The system crawls web pages, builds an inverted index, and ranks results using a combination of Vector Space Model (cosine similarity) and PageRank.

Architecture

1. Web Crawler

A polite web crawler that respects robots.txt and implements delays between requests. The crawler:

  • Starts from a set of seed URLs
  • Follows links to discover new pages
  • Checks robots.txt before accessing each domain
  • Implements configurable delays (politeness policy)
  • Extracts both content and link structure
  • Stores pages and link graph for offline processing

2. Indexer

Processes crawled HTML pages to build an inverted index:

  • Parses HTML to extract text content
  • Tokenizes and normalizes text (lowercase, remove stopwords)
  • Builds inverted index: term → {doc_id: frequency}
  • Calculates TF-IDF weights
  • Computes document lengths for normalization

3. PageRank Algorithm

Computes importance scores based on link structure:

  • Uses power iteration method
  • Damping factor: 0.85 (follows links 85% of the time)
  • Runs for 20 iterations to converge
  • Normalizes scores to [0, 1] range

4. Ranking Algorithm

Combines content-based and link-based scores using the formula:

score(d, q) = w1 × cosine_similarity(d, q) + w2 × pagerank(d)

where w1 + w2 = 1. The default weights are:

  • w1 = 0.6 (content relevance weight)
  • w2 = 0.4 (page importance weight)

Technical Details

Vector Space Model

Documents and queries are represented as vectors in a high-dimensional term space. The cosine similarity measures the angle between document and query vectors:

  • TF-IDF weighting scheme
  • Document length normalization
  • Cosine similarity for ranking

Text Processing

  • Tokenization using NLTK
  • Stopword removal (English)
  • Minimum term length: 2 characters
  • Case normalization

Implementation

  • Language: Python
  • Web Framework: Flask
  • HTML Parsing: BeautifulSoup
  • NLP: NLTK
  • Crawler: Custom implementation with robots.txt support

Features

  • Polite web crawling with robots.txt compliance
  • Efficient inverted index for fast retrieval
  • Combined content and link-based ranking
  • Clean web interface for query submission
  • Detailed scoring information (cosine + PageRank)

Dataset

The search engine indexes a minimum of 500 web pages. The current implementation focuses on Wikipedia pages for demonstration purposes, but can be configured to crawl any domain.
